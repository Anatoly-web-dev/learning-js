Операторы в JavaScript

Операнд – то, к чему применяется оператор. Например, в умножении 5 * 2 есть два операнда: левый равен 5, а правый равен 2.

Унарный оператор применяется к одному операнду. Например, оператор унарный минус "-" меняет на противоположный знак числа

let x = 1; x = -x; alert(x); // - применили унарный минус, результат -1

Бинарный оператор применяется к двум операндам. Тот же минус существует и в бинарной форме:

let x = 1, y = 3; alert(y - x ); // - бинарный минус вычитает значения, результат 2

В JavaScript есть много различных операторов, они обозначают операцию, которую нужно произвести.

Например: *, /, +, -, = и т.д.

Отдельно стоит поговорить про математические операторы:

- Взятие остатка от деления % несмотря на обозначение, никакого отношения к процентам не имеет.

Результат число a % число b – это остаток от целочисленного деления числа a на число b.

alert(5 % 2); // - 1, остаток от деления 5 на 2 alert(8 % 3); // - 2, остаток от деления 8 на 3

- Возведение в степень ** в выражении число a ** степень b оператор ** умножает число a на само себя b раз.

alert(2 ** 3); // - 8 (2 * 2 * 2, 3 раза) alert(2 ** 4); // - 16 (2 * 2 * 2 * 2, 4 раза)

Возведение в степень ** работает и для нецелых чисел.

Сложение строк при помощи бинарного +

Операция, когда мы «склеиваем» несколько значений, называется конкатенацией и в JavaScript выполняется с помощью знака плюс +.

let name = 'Борис'; paragraph.textContent = 'Вас зовут' + name + '. Хорошего дня!'; console.log (paragraph.textContent); // - выведет: Вас зовут Борис. Хорошего дня!

Если бинарный оператор '+' применить к строкам, то он их объединяет в одну:

let s = "моя" + "строка"; alert(s); // - моястрока

Правило сложения строк :Если хотя бы один операнд является строкой, то второй операнд будет также преобразован в строку.

alert('1' + 2); // - получим "12" alert(2 + 2 + '1'); // - получим "41"

Сложение и преобразование строк — это особенность бинарного плюса +. Другие арифметические операторы работают только с числами и всегда преобразуют операнды в числа.

alert(6 - '2'); // - результат 4; '2' приводится к числу alert('6' / '2'); // - результат 3; оба операнда приводятся к числам

Приведение к числу, унарный +

Плюс + существует в двух формах: бинарной, которую мы использовали выше, и унарной.

Унарный плюс, то есть применённый к одному значению, + ничего не делает с числами. Но если операнд не число, унарный плюс преобразует это значение в число.

Примеры:

let x = 1; alert(+x); // - не влияет на числа; результат 1 let y = - 2; alert(+y); // - не влияет на числа; результат -2 alert(+true ); // - преобразует не числа в числа; результат 1 alert(+""); // - преобразует не числа в числа; результат 0

На самом деле это то же самое, что и Number (value), только короче.

let apples = "2"; let oranges = "3"; alert(apples + oranges); // - результат "23", так как бинарный плюс объединяет строки alert(+apples + +oranges); //- результат 5, т.к. оба операнда предварительно преобразованы в числа . Cначала выполнятся унарные плюсы, которые приведут строки к числам, а затем бинарный '+' их сложит.

Приоритет операторов

В JavaScript много операторов. Каждый оператор имеет соответствующий номер приоритета. Тот, у кого этот номер приоритета больше – выполнится раньше. Если приоритет одинаковый, то порядок выполнения – слева - направо.

Приоритет унарных операторов выше, чем соответствующихбинарных.

Таблица приоритетов операторов в JavaScript

У оператора присваивания = один из наименьших приоритетов. Именно поэтому, когда переменной что-либо присваивают, например, x = 2 * 2 + 1, то сначала выполнится арифметика, а уже затем произойдёт присваивание = с сохранением результата в x.

Присваивание = возвращает значение.

let a = b = c = 2 + 2; Такое присваивание работает справа налево. Сначала вычисляется самое правое выражение 2 + 2, и затем результат переменным слева: c, b, a. У всех переменных в конце будет одно значение 4. чтобы код читался легче, лучше разделять подобные конструкции на несколько строчек: c = 2 + 2; b = c; a = c;

Инкремент / декремент

Инкремент ++ увеличивает переменную на 1. Декремент ++ уменьшает переменную на 1:

let counter = 2; counter++; alert(counter); // - получим 3 counter = 5; counter--; alert(counter); // - получим 4

Инкремент / декремент можно применить только к переменной. Попытка использовать его на значении, типа 5 ++, приведёт к ошибке.

Когда оператор идёт после переменной — это «постфиксная форма»: counter ++. «Префиксная форма» — это когда оператор идёт перед переменной: ++counter. Обе эти инструкции делают одно и то же: увеличивают counter на 1. Разница будет только если будем использовать значение, которое возвращает ++ / --. Префиксная форма возвращает новое значение, в то время как постфиксная форма возвращает старое значение (до увеличения / уменьшения числа).

Операторы ++/-- могут также использоваться внутри выражений. Их приоритет выше, чем у большинства других арифметических операций.

let counter = 1; alert(2 * ++counter); // - получим 4 let counter = 1; alert(2 * counter++); // - получим 2, потому что counter++ возвращает "старое" значение. Лучше использовать стиль «одна строка – одно действие»: let counter = 1; alert(2 * counter); counter++;

Побитовые операторы

Побитовые операторы работают с 32-разрядными целыми числами на уровне их внутреннего двоичного представления.

Они используются редко, когда возникает необходимость оперировать с числами на очень низком (побитовом) уровне.

Оператор «запятая» (,) редко применяется и является одним из самых необычных.

  • И - AND / &;
  • ИЛИ - OR / |;
  • Побитовое ИСКЛЮЧАЮЩЕЕ ИЛИ - XOR
  • NOT(НЕ) - ^ / ~;
  • и т.д.

Операторы сравнения

В JavaScript бывавют следующие операторы сравнения: классические Больше, Меньше, Больше или равно, Меньше или равно, а также Равно == , Не равно !=, Тождественно равно ===, Тождественно не равно !== и др.

Все операторы сравнения возвращают значение логического типа:

true – означает «да», «верно», «истина» или false – означает «нет», «неверно», «ложь».

alert(2 > 1); // - получим true (верно) alert( 2 == 1 ); // - получим false (неверно) alert( 2 != 1 ); // - получим true (верно)

Результат сравнения можно присвоить переменной, как и любое значение.

Сравнение строк

Чтобы определить, что одна строка больше другой, JavaScript использует «алфавитный» или «лексикографический» порядок. Другими словами, строки сравниваются посимвольно.

Алгоритм сравнения двух строк довольно прост:

  1. Сначала сравниваются первые символы строк.
  2. Если первый символ первой строки больше / меньше, чем первый символ второй строки, то первая строка больше / меньше второй. Сравнение завершено.
  3. Если первые символы равны, то таким же образом сравниваются уже вторые символы строк
  4. Сравнение продолжается, пока не закончится одна из строк.
  5. Если обе строки заканчиваются одновременно, то они равны. Иначе, большей считается более длинная строка.

Сравнение 'Я' > 'А' завершится на первом шаге, тогда как строки 'Кот' > 'Код' будут сравниваться посимвольно:

В JavaScript имеет значение регистр символов. Заглавная буква "A" не равна строчной "a". Какая же из них больше? - Строчная "a", потому что строчные буквы имеют больший код во внутренней таблице кодирования (Unicode), которую использует JavaScript

Сравнение разных типов

При сравнении значений разных типов JavaScript приводит каждое из них к числу.

alert('2' > 1); // - вернёт true, строка '2' становится числом 2 alert('01' == 1 ); // - вернёт true, строка '01' становится числом 1

Логическое значение true становится 1, а false становится 0.

Строгое сравнение

Использование обычного сравнения == может вызывать проблемы.Например, оно не отличает 0 от false

alert(0 == false); // - вернёт true Та же проблема с пустой строкой: alert('' == false); // - вернёт true

Это происходит из-за того, что операнды разных типов преобразуются оператором == к числу. В итоге, и пустая строка, и false становятся нулём.

Оператор строгого равенства === проверяет равенство без приведения типов. Другими словами, если a и b имеют разные типы, то проверка a === b немедленно возвращает false без попытки преобразования их

alert( 0 === false ); // - вернёт false, т.к. сравниваются разные типы

Ещё есть оператор строгого неравенства !==, аналогичный !=. Оператор строгого равенства дольше писать, но он делает код более очевидным и оставляет меньше места для ошибок.

Сравнение с null и undefined

При строгом равенстве === эти значения различны, т.к. различны их типы.

alert(null === undefined); // - вернёт false

При нестрогом равенстве == эти значения равны друг другу и не равны никаким другим значениям.

alert(null == undefined); // - вернёт true

При использовании математических операторов и других операторов сравнения, значения null / undefined преобразуются к числам: null становится 0, а undefined становится NaN.

Странный результат сравнения null и 0 alert(null > 0); // - вернёт false alert( null == 0 ); // - вернёт false alert(null >= 0); // - вернёт true

Значение undefined несравнимо с другими значениями:

Относитесь очень осторожно к любому сравнению с undefined / null, кроме случаев строгого равенства ===.

5 > 4; // - вернёт очевидно true "ананас" > "яблоко"; // - вернёт false т.к. используется посимвольное сравнение "2" > "12"; // - вернёт true, т.к. эти значения сравниваются посимвольно, как строки. undefined == null; // - вернёт true т.к. значения null и undefined равны только друг другу при нестрогом сравнении. undefined === null; // - вернёт false, т.к. строгое сравнение разных типов null == "\n0\n"; // - вернёт false т.к. null равен только undefined null === +"\n0\n"; // - вернёт false т.к строгое сравнение разных типов.

Логические операторы

В JavaScript есть три логических оператора: || (ИЛИ), && (И) и ! (НЕ).

В алгебре логики рассматривается три основные операции:

Отрицание - оператор НЕ ! изменяет значение на противоположное.

Конъюнкция - оператор И &&. Результат операции будет истинным тогда и только тогда, когда все (два) операнды конъюнкции истинны. Конъюнкция останавливается при нахождении первого ложного значения.

Дизъюнкция - оператор ИЛИ || Результат операции будет ложным тогда и только тогда, когда все (два) операнды дизъюнкции ложны: При этом оператор ИЛИ возвращает то значение, на котором остановились вычисления, без преобразования к логическому типу. Один из наиболее распространенных способов использования этого свойства — задание значения по умолчанию, если нет установленного.

И две константы: 0 и 1.

условие: Если текущая страница главная и пользователь авторизован; (состоит из 2 условий) действие: Вывод информации о последних операциях.

Закон противоречия гласит, что два противоположных выражения не могут быть одновременно истинными.

Закон исключения третьего гласит, что два противоположных значения не могут быть одновременно ложными.

Закон двойного отрицания гласит, что двойное отрицание какого-то логического значения даёт само значение.

Законы де Моргана:

  • Отрицание конъюнкции есть дизъюнкция отрицаний;
  • Отрицание дизъюнкции есть конъюнкция отрицаний.
Конъюнкция: isAdmin && isActive Дизъюнкция: isAdmin || isActive Отрицание конъюнкции: !(isAdmin && isActive) Отрицание дизъюнкции: !(isAdmin || isActive) Конъюнкция отрицаний: !isAdmin && !isActive Дизъюнкция отрицаний: !isAdmin || !isActive

В соответствии с законами де Моргана:

  • !(isAdmin && isActive) имеет всегда те же значения, что и !isAdmin || !isActive;
  • !(isAdmin || isActive) имеет всегда те же значения, что и !isAdmin && !isActive

|| (ИЛИ)

Существует всего четыре возможные логические комбинации с булевыми значениями:

alert(true || true); // - вернёт true alert(false || true); // - вернёт true alert(true || false); // - вернёт true alert(false || false); // - вернёт false

Результат операций всегда равен true, за исключением случая, когда оба аргумента false. Если значение не логического типа, то оно к нему приводится в целях вычислений. Например, число 1 будет воспринято как true, а 0 – как false.

Обычно оператор || используется в if для проверки истинности любого из заданных условий.

let hour = 9; if (hour < 10 || hour > 18) { alert('Офис закрыт.'); } ------------------- let time = 12; let isWeekend = true; if (time < 10 || time > 18 || isWeekend) { alert('Офис закрыт.'); }

ИЛИ || находит первое истинное значение

При выполнении ИЛИ || с несколькими значениями: let result = value1 || value2 || value3, оператор || выполняет следующие действия:

  • Вычисляет операнды слева направо.
  • Каждый операнд конвертирует в логическое значение. Если результат true, останавливается и возвращает исходное значение этого операнда.
  • - Если все операнды являются ложными (false), возвращает последний из них.

Значение возвращается в исходном виде, без преобразования.

Другими словами, цепочка ИЛИ || возвращает первое истинное значение или последнее значение, если такое значение не найдено.

alert(1 || 0); // - вернёт 1 alert(true || 'no matter what'); // - вернёт true alert(null || 1); // - вернёт 1 (первое истинное значение) alert(null || 0 || 1); // - вернёт 1 (первое истинное значение) alert(undefined || null || 0); // - вернёт 0 (поскольку все ложно, возвращается последнее значение)

Применение оператора ||

1. Получение первого истинного значения из списка переменных или выражений.

Представим, что у нас имеется ряд переменных, которые могут содержать данные или быть null / undefined. let currentUser = null; let defaultUser = "John"; let name = currentUser || defaultUser || "unnamed"; alert(name); // - получим "John" – первое истинное значение Если бы и currentUser, и defaultUser были ложными, в результате был бы "unnamed"

2. Сокращённое вычисление.

Операндами могут быть как отдельные значения, так и произвольные выражения. ИЛИ || вычисляет их слева направо. Вычисление останавливается при достижении первого истинного значения. Этот процесс называется «сокращённым вычислением», поскольку второй операнд вычисляется только в том случае, если первого недостаточно для вычисления всего выражения. Это хорошо заметно, когда выражение, указанное в качестве второго аргумента, имеет побочный эффект, например, изменение переменной.

В приведённом ниже примере переменная не изменяется:

let x; true || (x = 1); alert(x); // - получим undefined, потому что (x = 1) не вычисляется Если бы первый аргумент имел значение false, то || приступил бы к вычислению второго и выполнил операцию присваивания ------------------- let x; false || (x = 1); alert(x); // - вернёт 1

Как мы видим, этот вариант использования || является "аналогом if". Первый операнд преобразуется в логический. Если он оказывается ложным, начинается вычисление второго. В большинстве случаев лучше использовать «обычный» if, чтобы облегчить понимание кода, но иногда это может быть удобно.

&& (И)

В традиционном программировании И возвращает true, если оба аргумента истинны, а иначе – false

alert(true && true ); // - вернёт true alert(false && true ); // - вернёт false alert(true && false ); // - вернёт false alert(false && false ); // - вернёт false let hour = 12; let minute = 30; if(hour == 12 && minute == 30) { alert('The time is 12:30'); } Как и в случае с ||, любое значение допускается в качестве операнда && if (1 && 0) { // вычисляется как true && false alert("не сработает, так как результат ложный"); }

И «&&» находит первое ложное значение

При нескольких подряд операторах И: result = value1 && value2 && value3; Оператор && выполняет следующие действия:

  • Вычисляет операнды слева направо.
  • Каждый операнд преобразует в логическое значение. Если результат false, останавливается и возвращает исходное значение этого операнда.
  • Если все операнды были истинными, возвращается последний операнд.

Другими словами, И возвращает первое ложное значение или последнее, если ничего не найдено.

Эти правила схожи с поведением ИЛИ, разница в том, что И возвращает первое ложное значение, а ИЛИ – первое истинное значение. Примеры:

Если первый операнд истинный, && возвращает второй: alert(1 && 0); // - вернёт 0 alert(1 && 5); // - вернёт 5 Если первый операнд ложный, && возвращает его. Второй операнд игнорируется alert(null && 5); // - вернёт null alert(0 && "no matter what"); // - вернёт 0 Можно передать несколько значений подряд. В таком случае возвратится первое «ложное» значение, на котором остановились вычисления. alert(1 && 2 && null && 3); // - получим null Когда все значения верны, возвращается последнее alert(1 && 2 && 3 ); // - получим 3

Приоритет оператора И && больше, чем ИЛИ ||, так что он выполняетсяраньше. Т.о., код: a && b || c && d по существу такой же, как если бы выражения && были в круглых скобках: (a && b) || (c && d).

! (НЕ)

Оператор (НЕ) ! принимает один аргумент и выполняет следующие действия:

Сначала приводит аргумент к логическому типу (true / false), затем возвращает противоположное значение. Например:

alert(!true); // - результат: false alert(!0); // - результат: true двойное НЕ !! используют для преобразования значений к логическому типу alert(!!"non-empty string"); // - результат: true alert(!!null); // - результат: false То есть первое НЕ преобразует значение в логическое значение и возвращает обратное, а второе НЕ снова инвертирует его. В конце мы имеем простое преобразование значения в логическое.

Есть немного более подробный способ сделать то же самое – встроенная функция Boolean()

alert(Boolean ("non-empty string")); // - вернет true alert(Boolean (null)); // - вернет false

Приоритет НЕ ! является наивысшим из всех логических операторов, поэтому он всегда выполняется первым, перед && или ||.

Оператор нулевого слияния (??)

Результат выражения a ?? b будет следующим:

a, если значение a определено,

b, если значение a не определено.

То есть оператор ?? возвращает первый аргумент, если он не null / undefined, иначе второй. (как из двух значений получить одно «определённое»).

Вот как можно переписать выражение result = a ?? b, используя уже знакомые нам операторы:

let result = (a !== null && a !== undefined) ? a : b;

Как правило, оператор ?? нужен для того, чтобы задать значение по умолчанию для потенциально неопределённой переменной.

если переменная user не определена, покажем модальное окно с надписью 'Аноним' let user; alert(user ?? "Аноним"); // - получим Аноним Если бы переменная user содержала любое значение, кроме null / undefined, то мы бы получили его. let user = "Иван"; alert(user ?? "Аноним"); // - получим Иван

Кроме этого, можно записать последовательность из операторов ??, чтобы получить первое значение из списка, которое не является null / undefined.

let firstName = null; let lastName = null; let nickName = "Суперкодер"; alert(firstName ?? lastName ?? nickName ?? "Аноним"); // - получим Суперкодер, первое определённое значение

|| возвращает первое истинное значение, а ?? возвращает первое определённое значение.

Проще говоря, оператор || не различает false, 0, пустую строку "" и null / undefined. Для него они все одинаковые, т.е. являются ложными значениями. Если первым аргументом для оператора || будет любое из перечисленных значений, то в качестве результата мы получим второй аргумент.

let height = 0; alert(height || 100); // - вернёт 100 alert(height ?? 100); // - вернёт 0 height || 100 проверяет, имеет ли переменная height ложное значение, что так и есть. Т.о. результатом является второй аргумент, т.e. 100. height ?? 100 проверяет, что переменная height содержит null / undefined, а поскольку это не так, то результатом является сама переменная height, т.е. 0.

Оператор ?? имеет низкий приоритет: 4. Т.о., оператор ?? вычисляется до = и ?, но после большинства других операций, таких как +, * и т.д. Если нужно выбрать значение при помощи оператора ?? вместе с другими операторами в выражении, следует добавить круглые скобки (): Запрещено использовать вместе с || или && без явно указанных круглых скобок ().

Алгоритм бинарного поиска

  1. Определение значения элемента в середине структуры данных. Полученное значение сравнивается с ключом.
  2. Если ключ меньше значения середины, то поиск осуществляется в первой половине элементов, иначе — во второй.
  3. Поиск сводится к тому, что вновь определяется значение серединного элемента в выбранной половине и сравнивается с ключом.
  4. Процесс продолжается до тех пор, пока не будет найден элемент со значением ключа или не станет пустым интервал для поиска.

Бинарный поиск показывает хорошие результаты по скорости на больших упорядоченных списках. На неупорядоченных — бинарный поиск не работает (не имеет смысла шаг 2).

В зависимости от размера массива легко предугадать максимальное количество шагов поиска: 1 + log₂N, где N — число элементов. Разумеется, число требуется округлить до целого.

Логические операторы могут применяться к значениям любых типов. Полученные результаты могут иметь различный тип.
