Map, Set, JSON, Деструктуризация

Map

Map – это коллекция ключ / значение, как и Object. Но основное отличие в том, что Map позволяет использовать ключи любого типа.

Методы и свойства:

  • new Map( ) – создаёт коллекцию.
  • map.set (key, value) – записывает по ключу key значение value.
  • map.get (key) – возвращает значение по ключу или undefined, если ключ key отсутствует.
  • map.has (key) – возвращает true, если ключ key присутствует в коллекции, иначе false.
  • map.delete (key) – удаляет элемент по ключу key.
  • map.clear( ) – очищает коллекцию от всех элементов.
  • map.size – возвращает текущее количество элементов.

Map может использовать объекты в качестве ключей.

let john = {name: 'John'}; let visitsCountMap = new Map(); visitsCountMap.set (john, 123); // - объект john - это ключ для значения 123 в объекте Map console.log (visitsCountMap.get (john)); // - получим 123

Перебор Map

Для перебора коллекции Map есть 3 метода:

  • map.keys( ) – возвращает итерируемый объект по ключам,
  • map.values( ) – возвращает итерируемый объект по значениям,
  • map.entries( ) – возвращает итерируемый объект по парам вида [ключ, значение], этот вариант используется по умолчанию в for..of.

В отличие от обычных объектов Object, в Map перебор происходит в том же порядке, в каком происходило добавление элементов.

let recipeMap = new Map ([ ['огурец', 500], ['помидор', 350], ['лук', 50] ]); for (let vegetable of recipeMap.keys()) { // - перебор по ключам console.log(vegetable); // - выведет 'огурец', 'помидор', 'лук' } for (let amount of recipeMap.values()) { // - перебор по значениям console.log(amount); // - выведет 500, 350, 50 } for (let entry of recipeMap) { // - перебор по элементам в формате [ключ, значение], то же самое, что и recipeMap.entries() console.log(entry); // - выведет 'огурец', 500 ...(и так далее) }

Map имеет встроенный метод forEach( ), схожий со встроенным методом у массивов Array:

recipeMap.forEach ((value, key, map) => { // - выполняем функцию для каждой пары (ключ, значение) console.log(`${key}: ${value}`); // - выведет 'огурец':500 ... и так далее });

Object.entries(). Map из Object

При создании Map мы можем указать массив (или другой итерируемый объект) с парами ключ-значение для инициализации, как здесь:

массив пар [ключ, значение] let map = new Map ([ ['1', 'str1'], [1, 'num1'], [true, 'bool1'] ]); console.log (map.get ('1')); // - получим 'str1'

Если мы хотим создать Map из обычного объект, то нам поможет встроенный метод Object.entries (obj), который получает объект и возвращает массив пар ключ - значение.

let obj = { 'name': 'John', 'age': 30 }; let map = new Map (Object.entries (obj)); console.log (map.get ('name')); // - получим 'John' Object.entries( ) возвращает массив пар ключ-значение: [ ['name','John'], ['age', 30] ]. Это именно то, что нужно для создания Map.

Object.fromEntries: Object из Map

Есть метод Object.fromEntries( ), который делает противоположное: получив массив пар вида [ключ, значение], он создаёт из них объект:

let prices = Object.fromEntries([ ['banana', 1], ['orange', 2], ['meat', 4] ]); // - получим объект prices = { 'banana': 1, 'orange': 2, 'meat': 4 }

Мы можем использовать Object.fromEntries( ), чтобы получить объект из Map.

К примеру, у нас данные в Map, но их нужно передать в сторонний код, который ожидает обычный объект. let map = new Map(); map.set ('banana', 1); map.set ('orange', 2); map.set ('meat', 4); let obj = Object.fromEntries (map.entries()); // - получили обычный объект: obj = { banana: 1, orange: 2, meat: 4 }

Вызов map.entries( ) возвращает итерируемый объект пар ключ - значение, как раз в нужном формате для Object.fromEntries( ).

Это то же самое, так как Object.fromEntries( ) ожидает перебираемый объект в качестве аргумента, не обязательно массив. А перебор map как раз возвращает пары ключ - значение, так же, как и map.entries( ). Так что в итоге у нас будет обычный объект с теми же ключами - значениями, что и в map.

Set

Объект Set – это особый вид коллекции: «множество» значений (без ключей), где каждое значение может появляться только один раз. Основные методы объекта Set:

  • new Set (iterable obj) – создаёт Set, и если в качестве аргумента был передан итерируемый объект (массив), то копирует значения этого итерируемого объекта (массива) в новый Set.
  • set.add (value) – добавляет значение (если оно уже есть, то ничего не делает), возвращает тот же объект set.
  • set.delete (value) – удаляет значение, возвращает true, если value было на момент вызова в множестве , иначе вернёт false.
  • set.has (value) – возвращает true, если значение присутствует в множестве, иначе вернёт false.
  • set.clear() – удаляет все имеющиеся значения.
  • set.size – возвращает количество элементов в множестве.

Основная «изюминка» – это то, что при повторных вызовах set.add( ) с одним и тем же значением ничего не происходит, за счёт этого как раз и получается, что каждое значение появляется один раз.

Например, мы ожидаем посетителей, и нам необходимо составить их список. Но повторные визиты не должны приводить к дубликатам. Каждый посетитель должен появиться в списке только один раз. let set = new Set(); let john = { name: 'John'}; let pete = { name: 'Pete'}; let mary = { name: 'Mary'}; // считаем гостей, некоторые приходят несколько раз set.add (john); set.add (pete); set.add (mary); set.add (john); set.add (mary); console.log (set.size); // - получим 3 for (let user of set) { console.log (user.name); // - set сохранит только 3 уникальных значения: 'John', 'Pete', 'Mary' }

Мы можем перебрать содержимое объекта set как с помощью метода for..of, так и используя forEach( ):

let set = new Set (['апельсин', 'яблоко', 'банан']); for (let value of set) { console.log(value); } // - то же самое с forEach( ): set.forEach ((value, valueAgain, set) => { console.log(value); });

Set имеет те же встроенные методы, что и Map:

  • set.values( ) – возвращает перебираемый объект для значений,
  • set.keys() – то же самое, что и set.values(), присутствует для обратной совместимости с Map,
  • set.entries() – возвращает перебираемый объект для пар вида [значение, значение], присутствует для обратной совместимости с Map.

Для простых объектов доступны следующие методы:

  • Object.keys (obj) – возвращает массив ключей.
  • Object.values (obj) – возвращает массив значений.
  • Object.entries (obj) – возвращает массив пар [ключ - значение].

Некоторые отличия в синтаксисе обычного объекта и коллекции Map:

  • Вызов map.keys( ) возвращает перебираемый объект
  • Вызов Object.keys (obj) возвращает «реальный» массив, а не синтаксис obj.keys( )

JSON

Для того чтобы передать написанный на JavaScript объект языку с другими правилами составления объекта используется сериализация — перевод какой-либо структуры объектов в строку.

Популярный формат сериализации называется JSON («джейсон»), что означает JavaScript Object Notation или «текстовый формат обмена данными». JSON может быть сохранен в собственном файле с расширением .json и своим MIME-типом application / json и доступен для создания в редакторах кода (есть по умолчанию в JavaScript). Преимущества файлов JSON заключаются в скорости работы и загрузки с сервера, что позволяет использовать их в качестве баз данных.

Все имена свойств должны быть заключены в двойные кавычки "". Допускаются только простые значения в данных — без вызовов функций, привязок к чему либо и вычислений. Комментирование кода в JSON !не допускаются.

В коде файла JSON открытие глобального объекта всегда происходит без объявления его наименования, только фигурными скобками {} объекта или квадратными скобками [] массива:

Отличие синтаксиса, объекта JSON и обычного объекта:

объект JSON:

{

"Свойство": 'значение', } --------------- Если требуется создать JSON не в отдельном файле, а прямо в JavaScript , требуется написать объект с одиночными кавычками ' ' до и после фигурных скобок { }. let objJson = '{ "name": 'Aleksei', }'; --------------- объект JavaScript: let obj = { ключ: 'значение', }

Отличие массива JSON в том, что его значения идут упорядоченно по ячейкам, без имён свойств. Значения могут быть любыми.

[ 'Значение первой ячейки массива', 'Значение второй ячейки массива', 'Значение третьей ячейки массива' ]

Массивы JSON применяются, когда нужно передать определённое количество значений, без обращения отдельно к какому-то конкретному.

Объекты JSON могут вкладываться в массивы JSON и наоборот.

Несколько объектов в одном массиве: [ { "Свойство 1 объекта": 'значение' }, // - запятая для дополнительных вложенных объектов { "Свойство 2 объекта": 'значение' строки, "Ещё одно свойство 2 объекта": 'значение', } ] Дополнительные вложенные объекты и их свойства перечисляются через запятую (,). --------------- Массив JSON в объекте JSON объявляется через свойство: { "Свойство для массива": ['ячейка 1', 'ячейка 2',] }

!!! Важно! На один файл JSON может проходиться только по одному внешнему объекту JSON, либо массиву JSON. Два и более внешних объектов JSON или массивов JSON в одном файле JSON недопустимы.

JSON может принимать числа, boolean, строки, null, другие объекты и массивы.

Если свойство состоит из нескольких слов, то его указывают, начиная каждое новое слово с большойБуквы или через_нижнее_подчёркивание.

Метод stringify для перевода объекта в JSON

Метод JSON.stringify используется для перевода объекта в JSON внутри JS-файлов.

Для того чтобы перевести данный объект в формат JSON, метод stringify( ) нужно вложить внутрь новой переменной:

let json = JSON.stringify(obj);

Вывод значений обычного объекта в alert( ) невозможен. С JSON это можно сделать, так как идёт преобразование в строковый тип данных.

Если вывести в консоль переменную с JSON, то она не будет являться объектом.

При преобразовании через stringify( ) все функции-методы, undefined и Symbol(); будут проигнорированы.

Вложенные объекты также переводятся stringify( ) в JSON.

Функция replacer, перевод в JSON отдельных свойств

Метод stringify( ) помимо передачи объекта может передавать какие-либо его отдельные свойства. Это нужно для того, чтобы не задействовать дополнительные ресурсы сервера на передачу лишних данных.

Отдельные свойства объекта передаются с помощью встроенной функции в метод stringify( ) - replacer.

Располагается replacer через запятую (,) за названием объекта в круглых скобках () stringify и представляет собой квадратные скобки массива [], где перечисляются свойства, необходимые для перевода в формат JSON.

Передавать свойства в replacer нужно так же, как они написаны в изначальном объекте, но в одиночных кавычках '': 'свойство2'.

JSON.stringify (объект JS, [replacer]); let objJS = { свойство1: 'значение1', свойство2: 'значение2', свойство3: 'значение3', }; JSON.stringify(objJS, ['свойство2', 'свойство3']); Метод stringify в примере будет работать только со вторым и третьим свойством объекта, первое свойство не затронет. Получится JSON в одну строку: {"свойство2": 'значение2', ""свойство3": 'значение3'} В примере нужно отправить в JSON данные о пользователе: let user = { dateOfBirth:'2020-07-30', }; user.address = { city: 'Moscow', postalCode: 199000, }; // - Объект user содержит дату рождения dateOfBirth. address добавляется к объекту через отдельное присвоение свойства и включает в себя city и postalCode. let jsonUser = JSON.stringify(user, ['dateOfBirth', 'address']); // - В переменной jsonUser происходит передача stringify свойств dateOfBirth и address. console.log(jsonUser); - отобразит в браузере не то что ожидалось (нет города и кода), потому что replacer не выводит контент вложенных объектов, если он не указан явно. --------------- Для исправления ситуации нужно добавить внутрь квадратных скобок [] replacer свойства вложенного объекта address через запятую (,). let jsonUser = JSON.stringify(user, ['dateOfBirth', 'address', 'city', 'postalCode']); Теперь console.log (jsonUser); выведет указанные свойства вложенного объекта address

Параметр создания отступов space

Для форматирования строки можно использовать ещё один параметр, вложенный в stringify( ), — space (число).

JSON.stringify(объект, [replacer], space);

Параметр space располагается за replacer и представляет собой число пробелов для каждого свойства, указанного в stringify( ).

Метод parse для перевода JSON в объект

JSON.parse( ) нужен чтобы преобразовать JSON в объект. Этот процесс называется десериализация.

В примере jsonUser содержит строку JSON, полученную через stringify( ) из JS-объекта user: let user = { city: 'Moscow', name: 'Ann', age: 20, }; let jsonUser = JSON.stringify(user); let newJsObject = JSON.parse(jsonUser); // - Для применения метода parse( ) должен использоваться глобальный объект JSON. В круглых скобках () указывается переменная содержащая строку JSON. console.log(newJsObject); // - Результат: полноценный объект
