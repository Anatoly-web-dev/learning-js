Объекты в JavaScript

Объявление объекта

Объект — это неупорядоченная коллекция свойств, где каждое свойство состоит из имени ключа и ассоциированного значения с ним. Объекты в JavaScript можно называть ассоциативными массивами, так как они состоят из пары ключ-значение. Объекты предназначены для хранения сложных структур данных.

Объекты используются для хранения коллекций различных значений и более сложных сущностей.

К имени объекта присваивается литерал — заключённый в фигурные скобки {} список свойств со значениями.

По-другому свойство называется ключом объекта. Свойство и значение образуют пару «ключ:значение». Название ключа используется в функциях для вызова закреплённого значения, за ключом.

Объект может быть создан с помощью фигурных скобок {…} с необязательным списком свойств. Свойство – это пара «ключ: значение», где ключ – это строка (также называемая «именем свойства»), а значение может быть чем угодно. По ключу можно найти, удалить или добавить в значение что-либо.

let user = {}; // - Пустой объект синтаксис литерал объекта.

let user = { // - создаем объект. Если в объекте несколько свойств, то они перечисляются через (,). name: 'John', // - под ключом name хранится значение 'John'. "likes birds": true // - ключ состоящий из нескольких слов должен быть в кавычках (''). user.name = 'Pete'; // - Объект, объявленный через const, может быть изменён. Определение const выдаст ошибку только если мы присвоим переменной другое значение: user='новое значение'. };

Для обращения к свойствам используется запись «через точку»:

alert (user.name); // - John user.isAdmin = true; // - добавим новое свойство в объект (за пределами объекта) с логическим значением delete user.age; // - для удаления свойства мы можем использовать оператор delete alert (user['name']); // - мы также можем использовать для вызова свойства объекта квадратные скобки и кавычки ['свойство'] user.address = { // - добавляем вложенное объект, как свойство объекта. city: 'Москва', street: 'Ленина', }

Последнее свойство объекта может заканчиваться запятой (,). Такой подход упрощает добавление, удаление и перемещение свойств, так как все строки объекта становятся одинаковыми.

Объекты могут иметь вложенность. Т.е. внутри одного объекта может быть другой объект.

let userInfo = { age: 30, name: 'Петр', address: { // - создаем объект внутри другого объекта city: 'Москва' street: 'Ленина', }, } console.log(userInfo. address. street); // - получаем доступ к вложенному значению ключа street delete userInfo.address.city; // - удаляем вложенное свойство console.log (userInfo.address); // - выводим в консоль вложенный объект

Квадратные скобки

Для свойств, имена которых состоят из нескольких слов, доступ к значению 'через точку' не работает:

user.likes birds = true, // - это вызовет синтаксическую ошибку. Для таких случаев существует альтернативный способ доступа к свойствам через квадратные скобки [ ]. let user = {}; user["likes birds"] = true; // - присваивание значения ключу, имя которого состоящему из нескольких слов

Квадратные скобки [] также позволяют обратиться к свойству, имя которого может быть результатом выражения. Например, имя свойства может храниться в переменной:

let key = 'likes birds'; user[key] = true; // - то же самое, что и user['likes birds'] = true; Здесь переменная key может быть вычислена во время выполнения кода или зависеть от пользовательского ввода. После этого мы используем её для доступа к свойству. let user = { name: 'John', age: 30 }; let key = prompt('Что вы хотите узнать о пользователе?', 'name'); alert (user[key]); // - выведет John (если ввели "name"), доступ к свойству через переменную

Передача из переменной имени ключа

let like = 'your like JavaScript'; // - создаем переменную для (ключа объекта) let person = { // - создаем объект [like]: true, // - передаем имя свойства объекта из ранее созданной переменной like в квадратных скобках [] 0: 'Петя', } console.log (person1[0]); // - смотрим значение ключа 0. Если имя ключа является числом (тип number), чтобы обратиться к нему используют [] console.log (person[like]); // - смотрим значение ключа like

В большинстве случаев, когда имена свойств известны и просты, используется запись через точку (.). Если же нам нужно что-то более сложное, то мы используем запись через квадратные скобки [].

При вызове свойства через квадратные скобки [] обязательно должны присутствовать кавычки ''.

Для имен свойств объекта нет никаких ограничений (Даже можно использовать зарезервированные JS слова). Они могут быть в виде строк или символов (специальный тип для идентификаторов). Все другие типы данных будут автоматически преобразованы к строке.

Проверка существования свойства, оператор in

Cуществует специальный оператор in для проверки существования свойства в объекте.

Синтаксис: 'key' in object let user = { name: 'John', age: 30 }; alert('age' in user); // - результат true, свойство age есть в объекте user alert('отсутствующее свойство' in user); // - вернёт false, указанное свойство не существует в объекте user Cлева от оператора in должно быть имя свойства. Обычно это строка в кавычках. Если мы опускаем кавычки '', это значит, что мы указываем переменную, в которой находится имя свойства. Например: let user = { age: 30 }; let key = 'age'; alert(key in user); // - выведет true, имя свойства было взято из переменной key

Цикл for...in

Для перебора всех свойств объекта используется цикл for..in. Этот цикл отличается от цикла for(;;).

Синтаксис: for (key in object) { // - тело цикла выполняется для каждого свойства объекта } выведем все свойства объекта user: let user = { name: 'John', age: 30, isAdmin: true }; for (let key in user) { alert(key); // - выведет все ключи: name, age, isAdmin alert (user[key]); // - выведет все значения: John, 30, true }

Все конструкции for позволяют нам объявлять переменную внутри цикла как, например, let key здесь. Кроме того, мы могли бы использовать другое имя переменной. Например, часто используется вариант for (let prop in obj).

Cвойства объекта упорядочены особым образом: свойства с целочисленными ключами сортируются по возрастанию, остальные располагаются в порядке создания.

Целочисленное свойство означает строку, которая может быть преобразована в целое число и обратно без изменений. Т.e. '49' – это целочисленное имя свойства, потому что если его преобразовать в целое число, а затем обратно в строку, то оно не изменится. А вот свойства '+49' или '1.2' таковыми не являются.

С другой стороны, если ключи не целочисленные, то они перебираются в порядке создания.

Тип данных Symbol

Символ» представляет собой уникальный идентификатор. Создаются новые символы с помощью функции Symbol( )

let id = Symbol();

Символы гарантированно уникальны. Символы не преобразуются автоматически в строки.

let id = Symbol('id'); // - При создании символу можно дать описание ('id'). Описание – это просто метка, которая ни на что не влияет. alert(id); // - Вызовет ошибку!, потому что строки и символы – принципиально разные типы данных и не должны неконтролируемо преобразовываться друг в друга. alert (id.toString()); // - необходимо явно преобразовать его с помощью метода toString()

Символы позволяют создавать скрытые свойства объектов, к которым нельзя нечаянно обратиться и перезаписать из других частей программы.

let user = { // - создаем объект id: 35, // добавляем свойсво объекта с помощью ключа-строки id, присваиваем ему значение 35 }; let id = Symbol('id'); // - создаем ключ-символ с таким же именем id user[id] = 30; // - добавляем новое свойсво объекта с помощью ключа-символа id с тем же именем, что и у ранее созданного ключа-строки id, присваиваем ему значение 30 console.log (user['id']); // - обращаемся к ключу-строке id, результат 35 console.log (user[id]); // - обращаемся к ключу-символу id, результат 30 Используя ключ-символ id мы обезопасили код от перезаписи значения объекта с одинаковыми именами ключей

Свойства объекта, чьи ключи – символы, не перебираются! циклом for..in.

let id = Symbol('id'); let user = { name: 'Вася', age: 30, [id]: 123 }; for (let key in user) { alert(key); // - выведет все ключи объекта: name, age. Ключа-символа id нет среди них } alert (user[id]) // - хотя прямой доступ по символу id работает

А вот Object.assign, в отличие от цикла for..in, копирует и строковые и символьные свойства.

Копирование объектов и ссылки

Объекты хранятся и копируются по ссылке. Примитивные типы: строки, числа, логические значения – присваиваются и копируются по значению. Объекты ведут себя иначе. Переменная хранит не сам объект, а его адрес в памяти, другими словами ссылку на него.

Сам объект хранится где-то в памяти. А в переменной лежит ссылка на эту область памяти. Когда переменная объекта копируется – копируется ссылка, сам же объект не дублируется.

let user = { name: 'Иван' }; let admin = user; // - копируется по ссылке. Теперь у нас есть две переменные, каждая из которых содержит ссылку на один и тот же объект // - Мы можем использовать любую из переменных для доступа к объекту и изменения его содержимого: let user = { name: 'Иван' }; admin.name = 'Петя'; // - изменено по ссылке из переменной admin alert (user.name); // - выведет 'Петя', мы можем изменять значение по ссылке из переменной user Приведённый выше пример демонстрирует, что объект только один.

Клонирование и объединение объектов, Object.assign

Для дублирования (клонирования, создания независимой копии) объекта мы можем использовать метод Object.assign( ).

Object.assign() не делает глубокое клонирование объекта, если внутри копируемого объекта есть свойство, значение которого не является примитивом, оно будет передано по ссылке.

Синтаксис: Object.assign(куда (объект), что (свойство1), что(свойство2), ....) let userInfo = { name: 'Петя', age: 55, }; let user = Object.assign ({}, userInfo); // - копируем все значения объекта userInfo в новый объект user. user.age = 18;// - меняем значение свойства age у нового объекта user console.log (userInfo.age); // - убедились, что это теперь два разных объекта с разными значениями ключа age. console.log (user.age); // - убедились, что это теперь два разных объекта с разными значениями ключа age.

Мы можем объединить несколько объектов в один объект:

let user = { name: 'Иван', } let object1 = { canView: true } let object2 = { canEdit: true } Object.assign (user, object1, object2); // - копируем все свойства из object1 и object2 в user // - теперь user = {name: "Иван", canView: true, canEdit: true}

Если принимающий объект user уже имеет свойство с таким именем, оно будет перезаписано:

let user = { name: 'Иван' } Object.assign (user, {name: 'Пётр'}); alert (user.name); // - теперь user = { name: 'Пётр'}

Мы также можем использовать Object.assign() для замены for..in на простое клонирование:

let user = { name: 'Иван', age: 30 } let clone = Object.assign ({}, user); - скопируем все свойства объекта user в новый пустой объект clone.

Еще можем использовать Object.assign для добавления новых свойств в объект.

let user = { name: 'Иван', age: 30 } Object.assign(user, {['like Javascript']: true, city: 'Москва'}); console.log(user); // - покажет объект с новыми свойствами.

Вложенное клонирование

Например, у нас есть объект, у него есть свойство sizes, значением которого является объект: Теперь при клонировании объекта недостаточно просто скопировать clone.sizes = user.sizes, поскольку user.sizes – это объект, он будет скопирован по ссылке. А значит объекты clone и user в своих свойствах sizes будут ссылаться на один и тот же объект: let user = { name: 'Иван', sizes: { height: 182, width: 50, } } let clone = Object.assign({}, user); alert (user.sizes === clone.sizes); // - выведет true, один и тот же объект, user и clone обращаются к одному sizes user.sizes .width++; // - меняем свойство в объекте user alert (clone.sizes .width); // - видим, что значение свойства sizes изменилось и у объекта clone Чтобы исправить это, мы должны в цикле клонирования делать проверку, не является ли значение user[key] объектом, и если это так – скопировать его структуру. Это называется глубокое клонирование.

Опциональная цепочка ( .? )

Опциональная цепочка ?. — это безопасный способ доступа к свойствам вложенных объектов, даже если какое-либо из промежуточных свойств не существует.

Опциональная цепочка ?. останавливает вычисление и возвращает undefined, если значение перед ?. равно undefined или null.

let user = { name: 'Федор', address: { city: 'Краснодар', street: 'Красная', } } console.log (user ?. address ?. house); // - пытаемся обратиться к несуществующему свойству вложенного объекта, в результате получим undefined, а не ошибку, так было бы, если бы не использовали опциональную цепочку ?.

Не злоупотребляйте опциональной цепочкой ?., если слишком часто использовать ?., ошибки могут замалчиваться там, где это неуместно, и их будет сложнее отлаживать.

Опциональная цепочка ?. работает только с объявленными переменными.

Опциональная цепочка ?. — это не оператор, а специальная синтаксическая конструкция, которая также работает с функциями и квадратными скобками []. Например, ?.( ) используется для вызова функции, которая может не существовать.

Синтаксис ?.['имя ключа'] также работает, если мы хотим использовать скобки [] для доступа к свойствам вместо точки .. Это позволяет безопасно считывать свойство из объекта, который может не существовать.

Также мы можем использовать ?. с delete. Мы можем использовать ?. для безопасного чтения и удаления, но не для записи.

delete user ?.name; // - удаляет user.name если пользователь существует.
