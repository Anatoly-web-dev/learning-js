Функции в JavaScript

Объявление функции

Чтобы не повторять один и тот же код во многих местах, придуманы функции. Функции являются основными «строительными блоками» программы.

Синтаксис объявления функции:

function имяФункции (параметры) { тело функции... } function showMessage() { alert('Всем привет!'); } showMessage(); showMessage(); // - Вызов showMessage() выполняет код функции. (Увидим сообщение дважды)

Одно из главных предназначений функций: избавлениеот дублирования кода. Если понадобится поменять сообщение или способ его вывода – достаточно изменить его в одном месте: в функции, которая его выводит.

Переменные, объявленные внутри функции, видны только внутри этой функции.

function showMessage() { let message = "Привет, я JavaScript!"; // - локальная переменная alert(message); } showMessage(); // - выведет сообщение Привет, я JavaScript! alert( message ); // - будет ошибка, т.к. переменная видна только внутри функции

Функция обладает полным доступом к внешним переменным и может изменять их значение.

let userName = 'Вася'; function showMessage() { userName = "Петя"; // - изменяем значение внешней переменной let message = 'Привет', + userName; alert(message); } alert(userName); // - выведет Вася перед вызовом функции. showMessage(); alert(userName); // - выведет Петя, значение внешней переменной было изменено функцией

Внешняя переменная используется только если внутри функции нет такой локальной. Если одноимённая переменная объявляется внутри функции, тогда она перекрывает внешнюю.

let userName = 'Вася'; function showMessage() { let userName = 'Петя'; // - объявляем локальную переменную let message = 'Привет', + userName; // - вернёт Привет, Петя alert(message); } showMessage(); // - функция создаст и будет использовать свою собственную локальную переменную userName alert(userName); // - вернёт Вася, переменная не изменилась. Функция не трогала внешнюю переменную userName

Переменные, объявленные снаружи всех функций называются глобальными. Глобальные переменные видимы для любой функции (если только их не перекрывают одноимённые локальные переменные). Желательно сводить использование глобальных переменных к минимуму.

Вложенная функция не может быть вызвана извне функции включения.

Параметры функции

Мы можем передать внутрь функции любую информацию, используя параметры.

function showMessage (from, text) { // - аргументы: from, text alert(from + ':' + text); } showMessage('Аня', 'Привет!'); // - получим Аня : Привет! showMessage('Аня', 'Как дела?'); // - получим Аня: Как дела? переданные значения копируются в локальные переменные from и text. Затем они используются в теле функции.

Параметр – это переменная, указанная в круглых скобках в объявлении функции. Аргумент – это значение, которое передаётся функции при её вызове. Мы объявляем функции со списком параметров, затем вызываем их, передавая аргументы.

В качестве аргумента можно передать любое значение: строки, числа, логические значения, массивы, объекты, а также другие функции.

Если параметр не указан, то его значением становится undefined.

Если мы хотим задать параметру text значение по умолчанию, мы должны указать его после = при объявлении функции: function showMessage (from, text = "текст не добавлен") { alert(from + ':' + text); } showMessage('Аня'); // - получим Аня: текст не добавлен // - Теперь, если параметр text не указан во время вызова функции, его значением будет 'текст не добавлен'

Возврат значения (return)

Для сохранения результата выполнения функции и передачи его для использования к другим функциям используется ключевое действие return.

Функция может вернуть результат, который будет передан в вызвавший её код.

function sum (a, b) { return a + b; } let result = sum(1, 2); alert(result); // - получим 3

Директива return может находиться в любом месте в тела функции. Как только выполнение доходит до этого места, функция останавливается, и значение возвращается в вызвавший её код (присваивается переменной result выше). Вызовов return может быть несколько.

Через return можно сделать возврат значения внутри блоков if else. Такое же правило применяется во вложенных функциях, если требуется сохранить результат для передачи другим внешним функциям.

function checkAge (age) { if (age > 18) { return true; } else { return confirm( 'А родители разрешили?' ); } } let age = prompt('Сколько вам лет?', 18); if (checkAge (age)) { alert('Доступ получен'); } else { alert('Доступ закрыт'); }

Результат функции с пустым return или без него – undefined

Никогда не добавляйте перевод строки между return и его значением

Имена функций

Функция – это действие. Поэтому имя функции обычно является глаголом. Функция должна делать только то, что явно подразумевается её названием. И это должно быть одним действием.

function showMessage() {...} // - показывает сообщение function getAge() {...} // - возвращает возраст (в каком-либо значении) function calcSum() {...} // - вычисляет сумму и возвращает результат function createForm() {...} // - создаёт форму (и обычно возвращает её) function checkPermission() {...} // - проверяет доступ, возвращая true / false

Функции должны быть короткими и делать только что-то одно. Если это что-то большое, имеет смысл разбить функцию на несколько меньших.

Function Expression & Function Declaration

Когда функция объявляется по названию, среда выполнения сохраняет функцию в памяти. Во время вызова функции создается ссылка на сохранённое название.

function название() { } вызов по названию();

Существует ещё один синтаксис создания функций, который называется Function Expression (Функциональные выражения).

Функции можно писать без имени для сокращения синтаксиса кода, когда используется функциональное выражение function expression:

let fun = function() {...}; fun();

Когда функция объявлена без имени, она называется анонимной функцией. Функция хранится в памяти, но среда выполнения не создает ссылку на неё автоматически, как это происходит с названием.

let sayHi = function() { // - объявляем переменную и присваиваем ей функцию без имени alert('Привет'); }; // - обязательно нужно ставить точку с запятой (;) sayHi(); // - вызываем функцию функция создаётся и явно присваивается переменной, как любое другое значение Мы можем скопировать функцию в другую переменную: function sayHi() { // - создали функцию и присвоили её значение переменной с именем func. alert('Привет'); } let func = sayHi; // - мы скопировали её значение в переменную func. нет круглых скобок () после sayHi. Если бы () были, то выражение func = sayHi() записало бы результат вызова sayHi() в переменную func, а не саму функцию sayHi. func(); // - выведет Привет, теперь функция может быть вызвана с помощью обеих переменных sayHi() и func(). sayHi(); // - выведет Привет

Функция объявленная внутри блока (например if) не видна за его пределами.

if (2 > 1) { function getSumm() { let sum = (1 + 2); console.log(sum); } } getSumm(); // - вызовет ошибку Правильный вариант: let getSumm; // - объявляем переменную за пределами блока if (2 > 1) { getSumm = function () { // - присваиваем переменной функцию let sum = (1 + 2); console.log(sum); } } getSumm(); // - вызываем эту функцию.

Функции – это значения. Они могут быть присвоены, скопированы или объявлены в любом месте кода.

Если функция объявлена как отдельная инструкция в основном потоке кода, то это Function Declaration.

Если функция была создана как часть выражения, то считается, что эта функция объявлена при помощи Function Expression.

Function Declaration обрабатываются перед выполнением блока кода. Они видны во всём блоке.

Функции, объявленные при помощи Function Expression, создаются только когда поток выполнения достигает их.

В большинстве случаев, когда нам нужно создать функцию, предпочтительно использовать Function Declaration, т.к. функция будет видима до своего объявления в коде. Это позволяет более гибко организовывать код и улучшает его читаемость.

В строгом режиме, когда Function Declaration находится в блоке {...}, функция доступна везде внутри блока. Но не снаружи него.

Стрелочные функции

Существует ещё один очень простой и лаконичный синтаксис для создания функций, он называется «стрелочные функции» (arrow functions). Такие функции требуется присваивать к переменной. Выглядит следующим образом:

let имя = (параметр1, параметр2, ... параметр-n) => выражение; let getSum = (a, b) => a + b; Эта стрелочная функция представляет собой более короткую форму: let sum = function(a, b) { return a + b; }; alert(sum (1, 2)); // - выведет 3

Если у нас только один аргумент, то круглые скобки () вокруг параметров можно опустить.

Если аргументов нет, круглые скобки () будут пустыми, но они должны присутствовать.

Стрелочные функции можно использовать так же, как и Function Expression. Например, для динамического создания функции:

let age = prompt('Сколько Вам лет?', 18); let welcome = (age < 18) ? ()=> alert('Привет!'): ()=> alert('здравствуйте'); welcome();

Стрелочные функции не имеют собственного контекста выполнения this, но наследуют его от пользовательских родительских функций, куда они вкладываются.

Стрелочные функции позволяют получить возврат без использования return, когда в теле функции есть только вычисления значений или текст.

Обычная анонимная функция берёт свой контекст this из объекта, для которого она вызывается. Стрелочная функция должна быть вызвана в пользовательской родительской функции, чтобы определить this.

Не стоит использовать стрелочные функции, если это снижает читаемость кода.

Объекты и стрелочные функции

Стрелочные функции могут использоваться как методы объекта:

let obj = { method:() => { console.log('abc'); } }; obj.method();

В методе объекта при использовании стрелочной функции будет пропущено объявление function и название функции.

Важно! Такой подход НЕ рекомендуется к использованию, так как в методах важен контекст выполнения this.

let phone = { brand: 'Nokia', model: 'N8', color: 'Red', getModel: ()=> { return this.model; // - вернёт undefined }, getColor: function () { return this.color; // - вернёт 'Red' } }; console.log (phone.getModel()); // - вернёт undefined console.log (phone.getColor()); // - вернёт 'Red'

Стрелочные функции умеют возвращать результат через return объекты без использования фигурных скобок {}.

let car = (color = 'black') => {fuelType:'diesel', type: 'crossover'}; // - неправильный возврат объекта: Для того чтобы указать интерпретатору кода, что имеется в виду именно объект, нужно заключить его в круглые скобки (): let car = (color = 'black') => ({fuelType:'diesel', type: 'crossover'}); // - правильный возврат объекта:

Многострочные стрелочные функции

Стрелочные функции, которые мы видели до этого, были очень простыми. Они брали аргументы слева от => и вычисляли и возвращали выражение справа.

Иногда нам нужна более сложная функция, с несколькими выражениями и инструкциями. Это также возможно, нужно заключить их в фигурные скобки {}. При этом важное отличие – в том, что в таких скобках для возврата значения нужно использовать return (как в обычных функциях).

Объявление дополнительной переменной, которая нe относится к аргументам, считается как множество действий и требует фигурных скобок {}.

Функции стрелок могут содержать другие функции стрелок или обычные функции.

let getSum = (a, b) => { let result = a + b; return result; // - если используем фигурные скобки {}, то нужно явно указать return }; alert(getSum (1, 2)); // - выведет 3
