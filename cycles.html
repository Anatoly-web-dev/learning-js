Циклы в JavaScript

При написании скриптов зачастую встаёт задача сделать однотипное действие много раз. Например, вывести товары из списка один за другим. Или просто перебрать все числа от 1 до 10 и для каждого выполнить одинаковый код.

Для многократного повторения одного участка кода предусмотрены циклы.

Цикл «while»

Цикл while имеет следующий синтаксис:

while (условие) { // - код ... // - также называемый "телом цикла" }

Код из тела цикла выполняется пока условие истинно.

Например, цикл ниже выводит i, пока i < 3: let i = 0; while (i < 3) { // - выводит 0, затем 1, затем 2 alert(i); i++; }

Одно выполнение тела цикла по-научному называется итерация. Цикл в примере выше совершает три итерации.

Если бы строка i++ отсутствовала в примере выше, то цикл бы повторялся (в теории) вечно. На практике, конечно, браузер не позволит такому случиться, он предоставит пользователю возможность остановить «подвисший» скрипт, а JavaScript на стороне сервера придётся «убить» процесс.

Любое выражение или переменная может быть условием цикла, а не только сравнение, условие while вычисляется и преобразуется в логическое значение.

let i = 3; while (i) { // - когда i будет равно 0, условие станет ложным, и цикл остановится alert(i); // - выведет 3, 2, 1 i--; // - уменьшает каждый раз i }

Фигурные скобки {…} не требуются для тела цикла из одной строки. Если тело цикла состоит лишь из одной инструкции, мы можем опустить фигурные скобки {…}:

let i = 3; while (i) alert(i--);

Цикл «do…while»

Проверку условия можно разместить под телом цикла, используя специальный синтаксис do ... while:

do { // - Цикл сначала выполнит тело цикла тело цикла... } while (условие); // - а затем проверит условие, и пока его значение равно true, цикл будет выполняться снова и снова. let i = 0; do { alert( i ); i++; } while (i < 3); Такая форма синтаксиса оправдана, если вы хотите, чтобы тело цикла выполнилось хотя бы один раз, даже если условие окажется ложным. На практике чаще используется форма с предусловием: while( ) { }.

Цикл «for»

Более сложный, но при этом самый распространённый цикл — цикл for.

Синтаксис: for (начало; условие; шаг) { тело цикла ... } Цикл ниже выполняет alert(i) для i от 0 до (но не включая) 3: for (let i = 0; i < 3; i++) { // - выведет 0, затем 1, затем 2 alert(i); }

Рассмотрим конструкцию for подробней:

начало let i = 0 выполняется один раз при входе в цикл условие i < 3 проверяется перед каждой итерацией цикла. Если оно вычислится в false, цикл остановится. тело alert (i) выполняется снова и снова, пока условие вычисляется в true. шаг i++ выполняется после тела цикла на каждой итерации перед проверкой условия.

В целом, алгоритм работы цикла выглядит так:

выполнить начало → (eсли условие == true → выполнить тело → выполнить шаг) → (eсли условие == true → выполнить тело → выполнить шаг) → (eсли условие == true → выполнить тело → выполнить шаг) → ... Т.е. начало выполняется один раз, а затем каждая итерация заключается в проверке условия, после которой выполняется тело и шаг.

Встроенное объявление переменной

В примере переменная счётчика i была объявлена прямо в цикле. Это так называемое «встроенное» объявление переменной. Такие переменные существуют только внутри цикла.

for (let i = 0; i < 3; i++) { alert(i); // - выведет 0, 1, 2 } alert(i); // - получим ошибку, нет такой переменной

Вместо объявления новой переменной мы можем использовать уже существующую:

let i = 0; for (i = 0; i < 3; i++) { // - используем существующую переменную alert(i); // - выведет 0, 1, 2 } alert(i); // - получим 3, переменная доступна, т.к. была объявлена снаружи цикла

Пропуск частей «for»

Любая часть for может быть пропущена. Для примера, мы можем пропустить начало, если ничего не нужно делать перед стартом цикла.

let i = 0; // - мы уже имеем объявленную i с присвоенным значением for (; i < 3; i++) { // - нет необходимости в начале alert(i); // - выведет 0, 1, 2 } Можно убрать и шаг: let i = 0; for (; i < 3;) { alert(i++); } Это сделает цикл аналогичным while(i < 3){...}. А можно и вообще убрать всё, получив бесконечный цикл: for (;;) { // - тело будет выполняться вечно } Точки с запятой ( ; ; ) обязательно должны присутствовать, иначе будет ошибка.

Директива break и continue

Обычно цикл завершается при вычислении условия в false. Но мы можем выйти из цикла в любой момент с помощью специальной директивы break.

Например, следующий код подсчитывает сумму вводимых чисел до тех пор, пока посетитель их вводит, а затем – выдаёт: let sum = 0; while(true) { let value = +prompt("Введите число", ''); if (!value) break; // - директива break полностью прекращает выполнение цикла и передаёт управление на строку за его телом, то есть на alert. sum += value; } alert(`Сумма: ${sum}`); Еще пример: while(true) { let user = prompt('Введите свое имя!'); if (user === 'Анатолий' || user === 'Инна' || user === null) break; // - цикл остановится когда будет выполнено одно из условий. }

Переход к следующей итерации: continue

Директива continue – «облегчённая версия» break. При её выполнении цикл не прерывается, а переходит к следующей итерации (если условие все ещё равно true).

Например, цикл ниже использует continue, чтобы выводить только нечётные значения:

for (let i = 0; i < 10; i++) { if (i % 2 == 0) continue; // - если true, то пропустить оставшуюся часть тела цикла alert(i); // - выведет 1, 3, 5, 7, 9 } Для чётных значений i, директива continue прекращает выполнение тела цикла и передаёт управление на следующую итерацию for (со следующим числом). Т.о. alert вызывается только для нечётных значений.

Директива continue позволяет избегать вложенности

Цикл, который обрабатывает только нечётные значения, мог бы выглядеть так: for (let i = 0; i < 10; i++) { if (i % 2) { alert(i); } }

С технической точки зрения он полностью идентичен. Действительно, вместо continue можно просто завернуть действия в блок if.

Однако мы получили дополнительный уровень вложенности фигурных скобок. Если код внутри if более длинный, то это ухудшает читаемость, в отличие от варианта с continue.

Нельзя использовать break / continue справа от оператора „?“

Обратите внимание, что эти синтаксические конструкции не являются выражениями и не могут быть использованы с тернарным оператором ?. В частности, использование таких директив, как break / continue, вызовет ошибку.

Метки для break/continue

Бывает, нужно выйти одновременно из нескольких уровней цикла сразу.

Например, в коде ниже мы проходимся циклами по i и j, запрашивая с помощью prompt координаты (i, j) с (0, 0) до (2, 2): for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { let input = prompt(`Значение на координатах (${i}, ${j})`, ''); // - Что если мы захотим перейти к Готово (ниже) прямо отсюда? } } alert('Готово!'); Нам нужен способ остановить выполнение цикла если пользователь отменит ввод. Обычный break после input лишь прервёт внутренний цикл, но этого недостаточно. Достичь желаемого поведения можно с помощью меток.

Метка имеет вид идентификатора с двоеточием : перед циклом:

labelName: for (...) { ... }

Вызов break метка в цикле ниже ищет ближайший внешний цикл с такой меткой и переходит в его конец.

outer: for (let i = 0; i < 3; i++) { for (let j = 0; j < 3; j++) { let input = prompt(`Значение на координатах (${i}, ${j})`, ''); if (!input) break outer; // - если пустая строка или Отмена, то выйти из обоих циклов // - сделать что-нибудь со значениями... } } alert('Готово!'); В примере выше это означает, что вызовом break outer будет разорван внешний цикл до метки с именем outer, и управление перейдёт со строки c break, к alert ('Готово!').

Итого:

Мы рассмотрели 3 вида циклов:

while – проверяет условие перед каждой итерацией.

do..while – проверяет условие после каждой итерации.

for (начало; условие; шаг) – проверяет условие перед каждой итерацией, есть возможность задать дополнительные настройки.

Чтобы организовать бесконечный цикл, используют конструкцию while (true). При этом он, как и любой другой цикл, может быть прерван директивой break.

Если на данной итерации цикла делать больше ничего не надо, но полностью прекращать цикл не следует – используют директиву continue.

Обе этих директивы поддерживают метки, которые ставятся перед циклом. Метки – единственный способ для break /continue выйти за пределы текущего цикла, повлиять на выполнение внешнего цикла.

Заметим, что метки не позволяют прыгнуть в произвольное место кода, в JavaScript нет такой возможности.
