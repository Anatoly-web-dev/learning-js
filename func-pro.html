Продвинутая работа с функциями в JS

Замыкание

Замыкание — это способность внутренней функции сохранять доступ к переменным и аргументам своей внешней родительской функции по цепочке области видимости, в том числе после того, как внешняя функция завершила выполнение.

function outer (outerArg) { // - Функция является так называемым лексическим окружением для вложенной функции. let outerVar; function inner() { // - Вложенная функция получит доступ к переменным и аргументам родительской функции outerArg. Т.е. inner получает доступ к outerVar и outerArg } }

Лексическое окружение — это то, что окружает функцию и содержит переменные, которые функция использует.

Замыкание используется для сохранения результата после каждого вызова функции.

Разберём более сложный классический пример:

function createCounter (counter = 0) { return function updateCounter() { console.log(counter++); } } let test = createCounter(5); С каждым вызовом test() в консоль выводится последовательно увеличивающийся на 1 аргумент counter: test(); test(); test();

У функции updateCounter( ) нет своего собственного аргумента counter, однако updateCounter( ) может использовать counter из своего лексического окружения, а именно из функции createCounter( ).

В функциональном выражении test( ) происходит сохранение внутренней функции updateCounter( ), возвращенной через return функцией createCounter( ).

Значение аргумента counter не сбрасывается до 0 при каждом вызове test( ).

Каждый раз к уже сохранённому значению counter++ прибавляется 1: чем больше произойдёт вызовов функции test( ), тем больше становится число записанное в аргументе.

При замыкании функций переменные и аргументы не исчезают из зоны видимости функций, а сохраняются внутри памяти.

Удалим замыкание, оставив только функциональное выражение, куда вносится функция updateCounter, увеличивающая аргумент counter через инкремент: let test = function updateCounter (counter = 0) { console.log(counter++); }; test(5); // - вернёт 5 test(); // - вернёт 0 test(); // - вернёт 0 // - Консоль отобразит: 5 -> 0 -> 0 * При отсутствии замыкания между вложенной и внешней функциями инкремент counter++ * не сохраняется для последующих вызовов, и test вызывается заново с указанным аргументом test( );

Функции-колбэки

JavaScript позволяет передать данные от одной функции к другой, где бы функции не располагались в коде, c помощью callback-функции.

Callback-функция передаётся другой функции в качестве аргумента во время вызова.

function main (name, number) { alert(`Привет! ${ name} Твой номер ${ number()}`); } function callback () { // - возвращает случайно сгенерированное число от 1 до 4. return parseInt (Math.random() * 4 + 1); } main(`Пётр, callback`); вызываем функцию с аргументами 'Пётр' и callback - функция-callback * Callback-функция, генерирующая числа, вызывается одновременно с функцией main. * Сначала сгенерируется число number, а затем main выведет число в alert.

callback принято передавать в качестве последнего аргумента функции, но это необязательное условие.

Функция задаёт вопрос question и, в зависимости от ответа пользователя, вызывает yes( ) или no( ): function ask (question, yes, no) { if (confirm( question)) yes(); else no(); } function showOk() { alert("Вы согласны."); } function showCancel() { alert("Вы отменили выполнение."); } функции showOk(), showCancel() передаются в качестве аргументов функции ask( ) ask("Вы согласны?", showOk, showCancel);

Аргументы функции ask ещё называют функциями-колбэками или просто колбэками.

Ключевая идея в том, что мы передаём функцию и ожидаем, что она вызовется обратно (от англ. «call back» – обратный вызов) когда-нибудь позже, если это будет необходимо. В нашем случае, showOk становится колбэком для ответа «yes», а showCancel – для ответа «no».

Функция – это значение, представляющее «действие». Мы можем передавать их из переменной в переменную и запускать когда захотим.

function calcSumm (num1, num2, more, less) { // - в основную функцию передаем 4 параметра let numSumm = num1 + num2; // - вычисляем сумму num1 и num2 if (numSumm > 3) { // - в зависимости от полученного значения выведется функция more( ) или less( ) more(); } else { less(); } } function showMoreMessage() { console.log('Больше чем 3'); // - выведет это сообщение, если условие выполнится } function showLessMessage() { console.log('Меньше чем 3'); // - выведет это сообщение, если условие выполнится } calcSumm(1, 5, showMoreMessage, showLessMessage); // - вызываем функцию с параметрами

Для того чтобы писать читаемые callback-функции, нужно следовать нескольким простым правилам:

  • - Комментировать код, что объяснит вам или другому разработчику, что происходит в программе.
  • - Писать именованные callback-функции вместо анонимных.
  • - Писать одну большую callback-функцию вместо нескольких маленьких с похожей логикой.

Рекурсия и стек

Рекурсия – это приём программирования, когда задача может быть разделена на несколько простых задач. Или когда задача может быть упрощена до несложных действий плюс простой вариант той же задачи.

В процессе выполнения задачи в теле функции могут быть вызваны другие функции для выполнения подзадач. Частный случай подвызова – рекурсия - когда функция вызывает сама себя.

Рекурсию используют, когда вычисление функции можно свести к её более простому вызову, а его – к ещё более простому и так далее, пока значение не станет очевидно.

Рекурсивное решение задачи короче, чем итеративное.

Рассмотрим два способа реализации функции pow(x, n), которая возводит x в натуральную степень n.

Итеративный способ: цикл for: function pow (x, n) { let result = 1; for (let i = 0; i < n; i++) { // - умножаем result на x n раз в цикле result *= x; } return result; } alert(pow (2, 3)); // - вернёт 8 Рекурсивный способ: упрощение задачи и вызов функцией самой себя: function pow (x, n) { if (n == 1) { return x; } else { return x * pow(x, n - 1); } } alert(pow (2, 3)); // - вернёт 8

Общее количество вложенных вызовов (включая первый) называют глубиной рекурсии. Для решения большого числа задач рекурсивный способ решения даёт более простой код, который легче поддерживать.

Один вызов функции имеет ровно один контекст выполнения, связанный с ним.

Когда функция производит вложенный вызов, происходит следующее:

  • Выполнение текущей функции приостанавливается.
  • Контекст выполнения, связанный с ней, запоминается в специальной структуре данных – стеке контекстов выполнения.
  • Выполняются вложенные вызовы, для каждого из которых создаётся свой контекст выполнения.
  • После их завершения старый контекст достаётся из стека, и выполнение внешней функции возобновляется с того места, где она была остановлена.

Любая рекурсия может быть переделана в цикл.

Остаточные параметры и оператор расширения

Вызывать функцию можно с любым количеством аргументов независимо от того, как она была определена. Лишние аргументы не вызовут ошибку.

Остаточные параметры могут быть обозначены через три точки (...). Мы можем положить первые несколько параметров в переменные, а остальные – собрать в массив. Все аргументы функции находятся под своими порядковыми номерами.

Остаточные параметры (...) собирают все остальные аргументы, поэтому бессмысленно писать что-либо после них. Это вызовет ошибку.

function showName(firstName, lastName, ...titles) { alert(firstName + ' ' + lastName); // - выведет Юлий Цезарь // - Оставшиеся параметры пойдут в массив, titles = ['Консул', 'Император'] alert (titles[0]); // - выведет 'Консул' alert (titles[1]); // - выведет 'Император' alert (titles.length); // - выведет 2 }

Оператор расширения похож на остаточные параметры – тоже использует (...), но делает противоположное. Когда (...) используется при вызове функции, он «расширяет» массив в список аргументов. Мы даже можем комбинировать оператор расширения с обычными значениями.

let arr1 = [3, 5, 1]; let arr2 = [8, 3, -8, 1]; Можем передать несколько массивов с параметрами в список аргументов: alert (Math.max (...arr1, ...arr2)); // - вернёт 8, оператор "раскрывает" массив

Как отличить их друг от друга:

  • Если (...) располагается в конце списка параметров функции, то это «остаточные параметры». Он собирает остальные неуказанные аргументы и делает из них массив. Остаточные параметры используются, чтобы создавать новые функции с неопределённым числом аргументов.
  • Если (...) встретился в вызове функции или где-либо ещё, то это «оператор расширения». Он извлекает элементы из массива. С помощью оператора расширения можно вставить массив в функцию, которая по умолчанию работает с обычным списком аргументов.

Каррирование

Каррирование – это трансформация функций таким образом, чтобы они принимали аргументы не как funcName(a, b, c), а как funcName(a) (b) (c)

Каррирование не вызывает функцию. Оно просто трансформирует её.

Для каррирования необходима функция с фиксированным количеством аргументов. Функцию, которая использует остаточные параметры, типа funcName (...args), так каррировать не получится.

function curry(funcName) { // - curry (f) выполняет каррирование return function(a) { return function(b) { return funcName(a, b); }; }; } // - использование function sum(a, b) { return a + b; } let curriedSum = curry(sum); alert(curriedSum (1)(2)); // - вернёт 3

Объект функции, NFE

В JavaScript функции – это объекты. Можно представить функцию как «объект делающий какое-то действие». Функции можно не только вызывать, но и использовать их как обычные объекты: добавлять / удалять свойства, передавать их по ссылке и т.д.

Объект функции содержит несколько полезных свойств. Например, имя функции нам доступно как свойство name. Также имена имеют и методы объектов.

function sayHi() { alert('Hi'); } alert (sayHi.name); // - выведет имя функции 'sayHi'

Cвойство length содержит количество параметров функции при её объявлении. Троеточие (...), обозначающее остаточные параметры, здесь как бы не считается.

Мы также можем добавить собственные свойства. Многие известные JavaScript-библиотеки искусно используют эту возможность.

Мы можем использовать функцию как объект, хранить в ней свойства, но они никак не влияют на выполнение функции. Переменные – это не свойства функции. Иногда свойства функции могут использоваться вместо замыканий. Изменить его могут только вложенные функции.

Named Function Expression или NFE – это термин для функционального выражения, у которого есть имя.

let sayHi = function funcName(who) { alert(`Hello, ${who}`); };

Добавление funcName после function не превращает объявление в Function Declaration, потому что оно все ещё является частью выражения присваивания. Добавление такого имени ничего не ломает. Функция все ещё доступна как sayHi( ):

Есть две важные особенности имени funcName, ради которого оно даётся:

  • Оно позволяет функции ссылаться на себя же.
  • Оно не доступно за пределами функции.
let sayHi = function funcName(who) { if (who) { alert(`Hello, ${who}`); } else { funcName('Guest'); // - Теперь всё в порядке } }; let welcome = sayHi; sayHi = null; welcome(); // - вернёт 'Hello, Guest' (вложенный вызов работает). Потому что имя funcName локальное и находится внутри функции. Теперь оно взято не снаружи (и недоступно оттуда).

Это не работает с Function Declaration. Зачастую, когда нам нужно надёжное внутреннее имя, стоит переписать Function Declaration на Named Function Expression.
